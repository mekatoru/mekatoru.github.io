案例项目：

Goal

学习拿到一个开源 repo 后如何拆地图：

  1. 先看 README，确认项目能做什么。
  2. 看目录结构，区分 inference、finetune、model、tools。
  3. 找推理入口，理解从 prompt 到 video 的路径。
  4. 找训练/微调入口，理解数据、模型、loss、optimizer 的路径。
  5. 找模型定义，确认 VAE、Transformer/DiT、scheduler、text encoder 在哪里。
  6. 看配置和脚本，而不是一开始就扎进所有源码。

Things To Notice

  1. README 提到 CogVideoX 有 Diffusers 和 SAT 两种使用路径。
  2. README 中列出不同模型的显存需求、分辨率、帧数和推理精度。
  3. README 提到 VAE slicing、tiling、CPU offload、quantization 等显存优化手段。
  4. 项目结构里单独有 finetune/，说明微调入口和推理入口通常分开。
  5. 真实项目会有很多工程代码，第一遍不要试图逐行读完。

First Questions
